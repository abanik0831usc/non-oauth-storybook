## Widget Screens

###### Screens are a mechanism by which users can take certain actions. For example, In **Search & Select** screen, they can select a provider. In **Authorization** screen they can enter their credentials.

| Widget Screens | Actions | Image view |
| :-:           | :-:            | :-: |
| **Search & Select** | This is the **landing screen** for Intuit Provider Selector. It let's the user select a provider which they desire to link with Intuit | <img src="https://files-emirs5o0r-abanik0831usc.vercel.app" alt="drawing" width="500"/>
| **Authentication** | In this screen, the user provides their **username** and **password** of the selected provider. | <img src="https://files-rn526qfte-abanik0831usc.vercel.app" alt="drawing" width="500"/>
| **Recaptcha** | Recaptcha screen helps us identify if the flow is generated by a human  | <img src="https://files-pclm4vqye-abanik0831usc.vercel.app" alt="drawing" width="500"/>
| **MFA (multi-factor authentication)** | Certain providers will require users to provide additional authorization details like MFA. this screen enables the user to provide such information  | <img src="https://files-lf78q51mv-abanik0831usc.vercel.app" alt="drawing" width="500"/>
| **Connecting** | Once the user have provided their account details, Intuit establishes a connection with the bank to fetch the details. During this process, a loading/connecting screen is shown to the user | <img src="https://files-jufc2d8j0-abanik0831usc.vercel.app" alt="drawing" width="500"/>
